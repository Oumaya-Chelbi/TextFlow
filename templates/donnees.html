Constitution du Dataset

De la collecte brute à la création des paires d'entraînement

Objectif

Notre modèle doit apprendre à trouver le meilleur endroit pour insérer un paragraphe dans un texte. Pour cela, nous avons besoin d'un dataset structuré contenant :

  • prev : le texte avant l'insertion
  • next : le texte après l'insertion
  • insert : le paragraphe à insérer
  • label : 1 si la position est correcte, 0 sinon

Sources des données

Pour construire notre dataset, nous avons utilisé trois sources majeures d'articles scientifiques et encyclopédiques :

Wikipédia - Articles encyclopédiques couvrant plusieurs domaines.
Téléchargement : dumps.wikimedia.org
arXiv - Articles scientifiques en accès libre, principalement en informatique.
Téléchargement : API arXiv
PMC (PubMed Central) - Articles biomédicaux en libre accès via la bibliothèque nationale de médecine des États-Unis.
Téléchargement : API E-utilities NCBI

1.2 Téléchargement des articles arXiv

Nous avons utilisé l'API arXiv pour récupérer des articles en informatique (catégorie cs.CL) :

# download_arxiv_pdfs.py
import requests
import feedparser

QUERY = "cat:cs.CL"
MAX_RESULTS = 650

url = f"http://export.arxiv.org/api/query?search_query={QUERY}&start=0&max_results={MAX_RESULTS}"
feed = feedparser.parse(url)

for entry in feed.entries:
    arxiv_id = entry.id.split("/")[-1]
    pdf_url = entry.links[1].href
    # Téléchargement du PDF
    r = requests.get(pdf_url)
    with open(f"arxiv_pdfs/{arxiv_id}.pdf", "wb") as f:
        f.write(r.content)

Puis conversion des PDF en texte :

# extract_arxiv_texts.py
import fitz  # PyMuPDF

def extract_text(pdf_path):
    doc = fitz.open(pdf_path)
    full_text = []
    for page in doc:
        text = page.get_text()
        if text.strip():
            full_text.append(text)
    doc.close()
    return "\n".join(full_text)

Étape 2 : Génération des exemples d'entraînement

Pour chaque source, nous avons généré des exemples où un paragraphe est retiré et doit être réinséré à sa position originale.

2.1 Traitement des articles Wikipédia

# generate_wiki.py
import json
import random

def generate_examples_from_text(title, text):
    paragraphs = [p.strip() for p in text.split("\n") if len(p.strip()) > 80]

    if len(paragraphs) < 5:
        return []

    examples = []
    idx = random.randint(0, len(paragraphs) - 1)
    removed = paragraphs[idx]
    truncated = paragraphs[:idx] + paragraphs[idx + 1:]

    example = {
        "source": "wikipedia",
        "title": title,
        "article_truncated": truncated,
        "paragraph_to_insert": removed,
        "true_position": idx
    }
    return [example]

Les scripts generate_arxiv.py  et generate_pmc.py suivent la même logique, adaptée au format de chaque source.

Étape 3 : Fusion et mélange des données

Pour créer un dataset homogène, nous avons fusionné toutes les sources puis mélangé aléatoirement les exemples :

# Fusion des trois sources
cat wiki_dataset.jsonl arxiv_dataset.jsonl pmc_dataset.jsonl > dataset_total.jsonl

# Mélange aléatoire des exemples
python shuffle_dataset.py
Pourquoi mélanger ?
Le mélange aléatoire (shuffling) est crucial pour éviter que le modèle n'apprenne des biais liés à l'ordre des sources. Sans mélange, le modèle pourrait mémoriser des patterns propres à une source plutôt que d'apprendre la tâche générale d'insertion.

Étape 4 : Split Train/Dev/Test

Nous avons divisé le dataset en trois parties selon la répartition classique :

  • 80% pour l'entraînement (train)
  • 10% pour la validation (dev)
  • 10% pour le test (test)
# split_dataset.py
import json
import random

with open("dataset_shuffled.jsonl") as f:
    data = [json.loads(line) for line in f]

random.shuffle(data)
n = len(data)

train = data[: int(0.8 * n)]
dev = data[int(0.8 * n): int(0.9 * n)]
test = data[int(0.9 * n):]

Étape 5 : Création des paires (prev, insert, next)

La dernière étape transforme chaque article en une série de paires d'insertion :

# build_pairs.py
import json
import argparse

def convert_dataset(input_file, output_file):
    with open(input_file, encoding="utf8") as f, open(output_file, "w", encoding="utf8") as out:
        for line in f:
            ex = json.loads(line)
            article = ex["article_truncated"]
            insert = ex["paragraph_to_insert"]
            gold = ex["true_position"]

            # Pour chaque position possible entre deux paragraphes
            for i in range(len(article) - 1):
                prev_p = article[i]
                next_p = article[i + 1]

                # Label 1 uniquement à la position correcte
                label = 1 if i == gold - 1 else 0

                pair = {
                    "prev": prev_p,
                    "next": next_p,
                    "insert": insert,
                    "label": label
                }
                out.write(json.dumps(pair, ensure_ascii=False) + "\n")
Pourquoi ce format ?
Notre modèle est entraîné à classifier si une position donnée (entre deux paragraphes) est le bon endroit pour insérer un paragraphe.

Exécution sur les trois splits :

python build_pairs.py --input train.jsonl --output train_pairs.jsonl
python build_pairs.py --input dev.jsonl --output dev_pairs.jsonl
python build_pairs.py --input test.jsonl --output test_pairs.jsonl

Étape 6 : Rétrécissement du dataset (downsampling)

Face aux contraintes de mémoire GPU et pour accélérer les phases d'expérimentation, nous avons dû réduire la taille de notre dataset tout en préservant son équilibre entre les classes. Le dataset initial contenait un très grand déséquilibre : pour chaque exemple positif (label 1), nous avions de nombreux exemples négatifs (label 0). Cela rendait l'entraînement très long et coûteux en ressources.

Nous avons donc appliqué un sous-échantillonnage (downsampling) en utilisant la méthode du réservoir sampling. Cette technique permet de sélectionner aléatoirement un nombre cible d'exemples par classe, sans avoir à charger tout le dataset en mémoire. Nous avons choisi les effectifs suivants :

  • Train : 20 000 exemples par classe (40 000 au total)
  • Dev (validation) : 2 000 exemples par classe (4 000 au total)
  • Test : 5 000 exemples par classe (10 000 au total)
Pourquoi cette réduction ?
- Temps d'entraînement : passage de plusieurs heures à 1 seule heure
- Mémoire GPU : adaptation aux limitations de nos GPUs
- Équilibre préservé : on conserve exactement le même nombre d'exemples pour les deux classes

Le script reservoir_sampling.py (disponible dans la section Scripts) implémente cette méthode et garantit la reproductibilité des résultats grâce à une graine aléatoire fixée (seed=42).

Répartition par source (des articles télécharger initialement)

  • Wikipédia : ~1000 articles
  • arXiv : 650 articles
  • PMC : 650 articles

Structure d'un exemple final

{
  "prev": "Le texte précédant la position d'insertion...",
  "next": "Le texte suivant la position d'insertion...",
  "insert": "Le paragraphe à insérer...",
  "label": 1
}

Chaque exemple correspond à une position d'insertion candidate. Le label indique si c'est la bonne position (1) ou non (0).
